1. HTML版本用户反馈

集成

把libUMFeedback.a，umFeedback.bundle，UMFeedback.h加入工程。

1. -(void)showFeedback:withAppkey:

使用HTML页面来展现用户反馈，取数据，发反馈，缓存等都是自动完成的。

在需要弹出用户反馈的地方添加代码:

[UMFeedback showFeedback:(UIViewController *)viewController withAppkey:(NSString *)appKey];

第一个参数(UIViewController *)viewController为HTML反馈弹出时的viewController,

反馈的界面是通过调用UIViewController的presentModalViewController:animated:方法实现的。

如果需要自定义CSS或数据项的话，可以修改umFeedback.bundle里的main.css或main.js。

目前HTML版的用户反馈支持了中文和英文界面，如果需要其他语言的支持，请拷贝chat_zh-Hans.html为chat_xx.html(xx为语言标记,ja等),修改html里的语言文本及main.js里CONFIG里的字符串常量。

如果需要在用户反馈数据里添加自定义字段，则需要修改main.js的submitForm函数。

Demo

//自定义字段，目前有提供了两个自定义Hash可用
//contact 用于传递联系方式，比如email,qq,phone等
//remark 用于传递备注信息，比如用户名,网址等等
// 参见下面两个例子,这里我们添加了网页表单id为email和user_name
var email = document.getElementById("email").value;
if (!!email) {
   result.contact.email = email;
}
var userName = document.getElementById("user_name").value;
if (!!userName) {
   result.remark.userName = userName;
}

2. -(void)showFeedback:withAppkey:dictionary:

功能同showFeedback:withAppkey:, 第三个参数dictionary的作用是传递数据至main.js 的 FeedbackJSON变量中。

开发者可能想在app中得到某个变量，然后发表反馈时自动填入到某个字段中，这时候就需要此方法。

Demo

[UMFeedback showFeedback:self withAppkey:UMENG_APPKEY dictionary:[NSDictionary dictionaryWithObject:[NSArray arrayWithObjects:@"a", @"b", @"c", nil] forKey:@"hello"]];

此时main.js中FeedbackJSON的值为{"hello":["a","b","c"]},

修改submitForm方法，把需要的值传入contact和remark既可。

2. Native数据接口

集成

把libUMFeedback.a，UMFeedback.h加入工程。

如果需要HTML版本的反馈，同时需要加入umFeedback.bundle。

1. +(UMFeedback )sharedInstance ### -(void)setAppkey:(NSString )appKey delegate:

如果需要使用数据接口，则需要得到一个UMFeedback单例,传入友盟的appkey和delegate。

Demo

UMFeedback *umFeedback = [UMFeedback sharedInstance];
[umFeedback setAppkey:UMENG_APPKEY delegate:self];

在需要的地方加上UMFeedbackDataDelegate,

Demo

@interface ViewController  : UIViewController <UMFeedbackDataDelegate>

在dealloc里设置umFeedback的delegate 为nil。

- (void)dealloc {
    umFeedback.delegate = nil;
}

数据接口有三个: get,post:,checkWithAppkey: 用于完成取，发和检查。

2. -(void)get

get 方法获取最新的用户反馈数据，异步方法，需要实现getFinishedWithError: delegate来完成回调。

如果error 为nil表示get正确完成，取实例的topicAndReplies属性即可获取最新的用户反馈数据。 常见的场景是当error 为nil时刷新UITableView。

topicAndReplies 为NSMutableArray，第一项为 topic,既用户发表的第一条主题，其余项为Replies,包括开发者回复的和用户回复的。两者的区别在type字段。

if ([reply objectForKey:@"type"] isEqualToString:@"dev_reply"]){
    NSLog(@"开发者回复");
}else{
    NSLog(@"用户回复");
}

如果想获取当次检查的newReplies(新回复),取实例的newReplies属性即可。

topicAndReplies 在取到数据后被缓存在NSUserDefaults,缓存这部分不需要自己去管理。

当次检查是指从上次get或check发生到现在为止，开发者给用户回复的数据，同样也是NSMutableArray类型，取这个数据可以完成通知提示，告诉用户有几条新回复。同样调用check方法可以达到同样的目的。

newReplies 大部分情况下都为nil,只在当次有新回复时才有值。

3. -(void)post:

post 方法发布用户反馈，异步方法，需要实现postFinishedWithError: delegate来完成回调。 用户发第一条主题和以后的回复都是使用post方法。

如果error 为nil表示post正确完成。

常见的场景是当error 为nil时告知用户发送成功。

post参数为NSDictionary *类型,其中可以包含的字段和说明见下面描述,其中contact和remark字段为自定义字段，用户可以传递自定义的内容。

feedback_dictionary 包含的有效key、含义、类型、取值如下：

gender,     性别,      NSString, "1"或"male"为男 "0"或"female" 为女
age_group,  年龄段,    NSString, "1"..."8",对应表如下
content,    反馈内容,  NSString,  长度不应超过255个字符
remark,     备注,      NSDictionary, 可以存放自定义的内容，比如评分等
contact,    联系方式，  NSDictionary, 可以存放和用户信息相关的内容，比如用户名等

"remark"和"contact"两个字段可供开发者传入自定义内容，这些内容会在网页上展现.

"age_group" 取值和年龄段对照表
"1" 小于18岁
"2" 18～24岁
"3" 25～30岁
"4" 31～35岁
"5" 36～40岁
"6" 41～50岁
"7" 51～59岁
"8" 大于60岁

demo

NSMutableDictionary *dictionary = [NSMutableDictionary dictionary];
[dictionary setObject:textFiled.text forKey:@"content"];
[dictionary setObject:@"2" forKey:@"age_group"];
[dictionary setObject:@"female" forKey:@"gender"];
[dictionary setObject:@[@"Good",@"VIP"] forKey:@"tags"];
NSDictionary *contact = [NSDictionary dictionaryWithObject:_emailField.text forKey:@"email"];
[dictionary setObject:contact forKey:@"contact"];
NSDictionary *remark = [NSDictionary dictionaryWithObject:_nameField.text forKey:@"name"];
[dictionary setObject:remark forKey:@"remark"];
[self.umFeedback postFeedback:dictionary];

4. + (void)checkWithAppkey:

checkWithAppkey: 方法用于检查是否有新回复，如果有新回复可以通知用户去查看，异步方法。

当checkWithAppkey:完成时需要去监听UMFBCheckFinishedNotification。

newReplies通过notification.userInfo被传递过来。

newReplies 大部分情况下都为nil,只在当次有新回复时才有值。

newReplies不为空可以提示用户有新回复, newReplies的长度就是新回复的条数。

Demo

[[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(umCheck:) name:UMFBCheckFinishedNotification object:nil];

----------------------------------------------------

- (void)umCheck:(NSNotification *)notification {
    UIAlertView *alertView;
    NSLog(@"notification = %@", notification.userInfo);
    if (notification.userInfo) {
        NSArray * newReplies = [notification.userInfo objectForKey:@"newReplies"];
        NSLog(@"newReplies = %@", newReplies);
        NSString *title = [NSString stringWithFormat:@"有%d条新回复", [newReplies count]];
        NSMutableString *content = [NSMutableString string];
        for (int i = 0; i < [newReplies count]; i++) {
            NSString * dateTime = [[newReplies objectAtIndex:i] objectForKey:@"datetime"];
            NSString *_content = [[newReplies objectAtIndex:i] objectForKey:@"content"];
            [content appendString:[NSString stringWithFormat:@"%d .......%@.......\r\n", i+1,dateTime]];
            [content appendString:_content];
            [content appendString:@"\r\n\r\n"];
        }

        alertView = [[UIAlertView alloc] initWithTitle:title message:content delegate:self cancelButtonTitle:@"取消" otherButtonTitles:@"查看", nil];
        ((UILabel *) [[alertView subviews] objectAtIndex:1]).textAlignment = NSTextAlignmentLeft ;

    }else{
        alertView = [[UIAlertView alloc] initWithTitle:@"没有新回复" message:nil delegate:nil cancelButtonTitle:@"取消" otherButtonTitles:nil];
    }
    [alertView show];
}

4. 常见问题

1.推送服务

SDK暂不支持APNS,只能通过定时执行checkWithAppkey:方法来实现。

2.程序未打开反馈界面时弹出新回复提示

可以在任何时候执行checkWithAppkey:方法，检查newReplies判断是否有新回复。 要做到这点需要在appDelegate里调用checkWithAppkey:。

3.我想发送反馈时带上用户的id和用户名

post:方法可以传递remark和contact这两个名称的NSDictionary *, 你需要把id和用户名等放到这两个字段里。

4.用你们的数据接口可以完成一个全功能、Native的反馈组件吗？

可以，Demo工程提供了一个简单例子，您可以在上面稍作扩展来实现。

5.Native数据接口如果做多语言支持？

数据接口和UI时完全分离的，您直接使用iOS自带的多语言方案即可，和数据接口基本无关。

6.Native数据接口有缓存吗？

有，生成sharedInstance 单例时， topicAndReplies 默认会从NSUserDefaults缓存取。

当执行get或checkWithAppkey:时缓存会同步更新。

7.我该选择HTML版本还是Native接口？

各有优缺，看自己的需求。

  • HTML版本的优点：定制容易，html，css，javascript修改简单，本身已经支持中英文的反馈界面。

HTML版本的缺点：速度慢，尤其是低端配置。

  • Native接口的优点：速度快，灵活，可以完全掌握UI实现。

Native接口的缺点：实现复杂，多语言支持等需要自己来做。

10.如何使用Native接口的字段？

构造反馈列表一般三个字段就够了。

  • type： 回复类型，user_reply(用户回复) dev_reply(开发者回复)
  • content： 反馈内容
  • datetime： 反馈时间

发表反馈时content为必填字段。
